Animate.css

Animación CSS de sólo agregar agua

animate.csses un montón de animaciones geniales, divertidas y de navegador cruzado para que las uses en tus proyectos. Excelente para enfatizar, páginas de inicio, controles deslizantes y, en general, simplemente añadir agua, maravilla.

Instalación

Instalar vía npm:

$ npm instala animate.css --save

o hilo

$ hilo agregar animate.css

Uso

Para usar animate.css en su sitio web, simplemente suelte la hoja de estilo en el documento <head>y agregue la clase animateda un elemento, junto con cualquiera de los nombres de animación. ¡Eso es! Tienes un elemento animado CSS. ¡Súper!

< cabeza >
  < link  rel = " hoja de estilo "  href = " animate.min.css " >
</ cabeza >

o usar una versión alojada de CDN por CDNJS

< cabeza >
  < link  rel = " hoja de estilo "  href = " https://cdnjs.cloudflare.com/ajax/libs/animate.css/3.7.0/animate.min.css " >
</ cabeza >

Es posible cambiar la duración de sus animaciones, agregar un retraso o cambiar la cantidad de veces que se reproducen:

.yourElement {
   animation-duration : 3 s ;
  animación-retraso : 2 s ;
  animación-iteración-cuenta : infinito ;
}

Uso con Javascript

Puedes hacer un montón de otras cosas con animate.css cuando lo combinas con Javascript. Un ejemplo simple:

const  elemento  =   documento . querySelector ( ' -elemento .my ' )
 elemento . classList . add ( ' animated ' , ' bounceOutLeft ' )

También puedes detectar cuando una animación termina:

const  elemento  =   documento . querySelector ( ' -elemento .my ' )
 elemento . classList . add ( ' animated ' , ' bounceOutLeft ' )

elemento . addEventListener ( ' animationend ' , function () { doSomething ()})

Puedes usar esta función simple para agregar y eliminar las animaciones:

function  animateCSS ( element , animationName , callback ) {
     const  node  =  document . querySelector (elemento)
     nodo . classList . add ( ' animated ' , animationName)

    function  handleAnimationEnd () {
         node . classList . remove ( ' animated ' , animationName)
         nodo . removeEventListener ( ' animationend ' , handleAnimationEnd)

        if ( typeof callback ===  ' function ' ) callback ()
    }

    nodo . addEventListener ( ' animationend ' , handleAnimationEnd)
}

Y úsalo así:

animateCSS ( ' .my-element ' , ' bounce ' )

// o 
animateCSS ( ' .my-element ' , ' bounce ' , function () {
   // Hacer algo después de la animación 
})

Tenga en cuenta que los ejemplos utilizan la constdeclaración de ES6 , eliminando el soporte para IE10 y algunos navegadores antiguos. Si lo prefiere, cambie consta las vardeclaraciones e IE10 y algunos navegadores antiguos obtendrán soporte (todavía tienen que proporcionar soporte de ClassList , así que haga su investigación ).

Ajuste de retraso y velocidad

Clase de retraso

Es posible agregar retrasos directamente en el atributo de clase del elemento, así:

< div  class = " animated bounce delay-2s " > Ejemplo </ div >
Nombre de la clase Tiempo de retardo
delay-2s 2s
delay-3s 3s
delay-4s 4s
delay-5s 5s

Nota : los retrasos predeterminados son de 1 segundo a 5 segundos solamente. Si necesita retrasos personalizados, agréguelo directamente a su propio código CSS.

Clase lenta, lenta, rápida y más rápida

Es posible controlar la velocidad de la animación agregando estas clases, como se muestra a continuación:

< div  class = " rebote animado más rápido " > Ejemplo </ div >
Nombre de la clase Tiempo de velocidad
slow 2s
slower 3s
fast 800ms
faster 500ms

Note: The animated class has a default speed of 1s. If you need custom duration, add it directly to your own CSS code.

Custom Builds

Animate.css is powered by gulp.js, which means you can create custom builds pretty easily. First of all, you’ll need Gulp and all other dependencies:

$ cd path/to/animate.css/
$ sudo npm install

Next, run gulp to compile your custom builds. For example, if you want only some of the “attention seekers”, simply edit the animate-config.json file to select only the animations you want to use.

"attention_seekers": {
  "bounce": true,
  "flash": false,
  "pulse": false,
  "shake": true,
  "headShake": true,
  "swing": true,
  "tada": true,
  "wobble": true,
  "jello":true
}

Accessibility

Animate.css supports the prefers-reduced-motion media query so that users with motion sensitivity can opt out of animations. On supported platforms (currently Firefox, OSX Safari and iOS Safari), users can select "reduce motion" on their operating system preferences and it will turn off CSS transitions for them without any further work required.
